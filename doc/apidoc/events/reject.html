PUT /api/v1/events/:id/reject
Mark a event as rejected

Supported Formats

json, xml

Errors

401 Unauthorized access
404 The requested resource was not found
406 The server cannot return data in the requested format
422 Unprocessable Entity: The change could not be processed because of errors on the data
500 Server crashed for some reason. Possible because of missing required params or wrong parameters

Examples

Response when the event was successfully rejected
PUT /api/v1/events/123/reject.json?auth_token=XXXXX&company_id=1
DATA: {
  reason: 'Please attach some photos of the event'
}

RESPONSE:
{
  success: true
  info: "Event successfully rejected"
  data: { }
}

Response when trying to reject a event that is already rejected
PUT /api/v1/events/123/reject.json?auth_token=XXXXX&company_id=1
DATA: {
  reason: 'Add the invoice for the expenses'
}
RESPONSE:
{
  success: false
  info: "Event cannot transition to rejected from rejected"
  data: { }
}

Params

Param name Description
auth_token
required

User's authorization token returned by login method


Value: Must be String
company_id
required

One of the allowed company ids returned by the “User companies” API method


Value: Must be a number.
id
required

Event ID


Value: Must be a number.
reason
required

Rejection reason (required when rejecting a event)


Value: Must be String
