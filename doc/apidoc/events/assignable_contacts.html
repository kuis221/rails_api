GET /api/v1/events/:id/assignable_contacts
Get a list of contacts+users that can be associated to the event as a contact

Also see events#add_contact.

Returns a list of contacts that can be associated to the event, including users but excluding those that are already associted.

The results are sorted by full_name.

Each item have the following attributes:

  • id: the user id

  • full_name: the user's full name

  • title: the user's title

  • type: indicates if the current item is a user or contact

Supported Formats

json, xml

Errors

400 Bad Request. he server cannot or will not process the request due to something that is perceived to be a client error.
401 Unauthorized access
404 The requested resource was not found
406 The server cannot return data in the requested format
422 Unprocessable Entity: The change could not be processed because of errors on the data
500 Server crashed for some reason. Possible because of missing required params or wrong parameters

Examples

An example with a user and a contact in the response

[

      "full_name": "Trinity Ruiz",
      "title": "Bartender",
      "type": "contact"

      "id": 268,
      "full_name": "Jonh Connor",
      "title": "Human Soldier",
      "type": "user"
  }
]

An example with a term search
